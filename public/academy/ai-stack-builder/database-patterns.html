Key Patterns

Pattern 1: Key-Value Brain

The brain_context table is a flexible key-value store. Keys use dot notation (like identity.name, session.active_work) for namespacing. Values are JSONB for maximum flexibility.

Pattern 2: Append-Only Memory

The agent_memory table stores every interaction. Never delete — always append. Use importance scores to prioritize retrieval. This is how AI agents build long-term memory.

Pattern 3: Event Streaming

The consciousness_stream is an event log. Every action the agent takes gets logged. Useful for debugging, auditing, and replaying agent behavior.
